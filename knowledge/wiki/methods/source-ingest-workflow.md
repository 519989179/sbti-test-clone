# 原始资料入库流程

## 核心结论
- 资料进来后，不能只是存；必须变成来源摘要 + 相关页面更新。

## 标准步骤
1. 将资料放入 `knowledge/raw/inbox/`
2. 阅读并判断类型
3. 生成 `wiki/sources/` 来源摘要页
4. 更新相关项目页 / 主题页 / 方法页 / 决策页
5. 更新 `wiki/index.md`
6. 记录 `wiki/log.md`
7. 原始资料移入对应分类目录

## 判断标准
- 如果是外部文章：优先产出来源摘要 + 主题页更新
- 如果是聊天结论：优先产出决策页或方法页
- 如果是书稿灵感：优先进入书稿项目页及相关论点页
- 如果是项目复盘：优先更新项目页与方法页

## 入库原则
- 一个来源能更新多个页面
- 尽量先更新旧页，再决定是否新建页
- 高价值信息必须带来源
- 页面写成“可复用认知”，别写成流水账
