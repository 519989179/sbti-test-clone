- 对插件/系统告警类问题，后续优先区分“功能是否受影响”和“日志是否干净”，避免为了不影响主流程的问题耗时过深。

  1. 能跑通
  2. 能稳定复现
  3. 内容/结果质量达标

- 对固定任务（如日更发布），继续把步骤沉淀成脚本 + 文档 + 定时任务，减少后续重复手工操作。
